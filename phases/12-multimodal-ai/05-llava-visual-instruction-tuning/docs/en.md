# LLaVA and Visual Instruction Tuning

> LLaVA (April 2023) is the most copied multimodal architecture on the planet. It replaced BLIP-2's Q-Former with a 2-layer MLP, swapped Flamingo's gated cross-attention for naive token concatenation, and trained on 158k visual instructions — generated by GPT-4 from text-only captions. Every practitioner who built a VLM between 2023 and 2026 built some LLaVA variant. LLaVA-1.5 added AnyRes. LLaVA-NeXT raised resolution. LLaVA-OneVision unified images, multi-image, and video with a single recipe. This lesson walks through the recipe, implements the projector, and explains why "simpler won."

**Type:** Build
**Languages:** Python (stdlib, projector + instruction template builder)
**Prerequisites:** Phase 12 · 02 (CLIP), Phase 11 (LLM Engineering — instruction tuning)
**Time:** ~180 min

## Learning Objectives

- Build a 2-layer MLP projector mapping ViT patch embeddings (dim 1024) to an LLM's embedding dimension (dim 4096).
- Walk through the LLaVA two-stage recipe: (1) projector alignment on 558k caption pairs, (2) visual instruction tuning on 158k GPT-4-generated conversations.
- Construct a LLaVA-format prompt with image token placeholders, system prompt, and user/assistant turns.
- Explain why the community migrated from Q-Former to MLP despite Q-Former's token budget advantage.

## The Problem

BLIP-2's Q-Former (Lesson 12.03) compresses an image to 32 tokens. Clean, efficient, good for benchmarks. But it has two problems.

First, the Q-Former is trainable, but its loss is not the final task. Stage 1 trains ITC+ITM+ITG. Stage 2 trains LM loss. The queries learn some intermediate representation that the LLM then has to decode. Information is lost at the bottleneck.

Second, the Q-Former is 188M parameters, and at LLaVA's 2023 scale, you have to co-design it with your target LLM. Swap the LLM, retrain the Q-Former. Swap the vision encoder, retrain. Each combination is a separate R&D project.

LLaVA's answer is embarrassingly simple: take the ViT's 576 patch tokens, pass each through a 2-layer MLP (`1024 → 4096 → 4096`), and dump all 576 into the LLM's input sequence. No bottleneck. No Stage 1 pretraining on weird objectives. Just an LM loss training the MLP directly.

Where does the data come from? LLaVA's second insight: use GPT-4 (text-only) to generate instruction data. Feed a COCO image's caption and bounding box data to GPT-4 and have it produce conversations, descriptions, and complex reasoning questions. 158k instruction-response pairs for free. No human annotation.

The result: a VLM trained on 8 A100s in one day that beats Flamingo on MMMU and ships an open checkpoint the community can extend. By late 2023, it had spawned 50+ forks.

## The Concept

### Architecture

LLaVA-1.5 at 13B:
- Vision encoder: CLIP ViT-L/14 @ 336 (frozen in Stage 1, optionally unfrozen in Stage 2).
- Projector: 2-layer MLP with GELU activation, `1024 → 4096 → 4096`.
- LLM: Vicuna-13B (later Llama-3.1-8B).

Forward pass for image + text prompt:

```
img -> ViT -> 576 patches of dim 1024
patches -> MLP -> 576 tokens of dim 4096
prompt: system + "<image>" placeholder + user question
replace <image> token with the 576 projected tokens
feed the full sequence to the LLM
decode response
```

The image occupies 576 tokens in the LLM's context. On a 2048 context, that leaves 1472 for text. On 32k context, it's a rounding error.

### Stage 1: Projector Alignment

Freeze ViT. Freeze LLM. Train only the 2-layer MLP. Dataset: 558k image-text pairs (LAION-CC-SBU). Loss: language modeling on captions conditioned on projected image tokens.

Batch 128 for a single epoch, done in a few hours. The projector learns to map ViT space to LLM space. No task-specific supervision.

### Stage 2: Visual Instruction Tuning

Unfreeze projector (still trainable). Unfreeze LLM (usually full fine-tune, sometimes LoRA). Train on 158k visual instructions.

The instruction data is the key trick. Liu et al. generated it like this:
1. Take a COCO image.
2. Extract textual descriptions (5 human captions + bounding box list).
3. Send to GPT-4 with three prompt templates:
   - Conversation: "Generate a multi-turn conversation between a user and an assistant about this image."
   - Detailed description: "Give a rich, detailed description of this image."
   - Complex reasoning: "Ask a question that requires reasoning about this image and answer it."
4. Parse GPT-4's output into (instruction, response) pairs.

None of this touches images directly — only text descriptions. GPT-4 hallucinates plausible image content. Some noise, but it works: 158k turns suffice to unlock conversational ability.

### Why the Community Copied This

- No Stage 1 specialized loss to tune. LM loss end-to-end.
- Projector trains in hours, not days.
- LLM swaps require only retraining the projector (LLaVA-Llama2, LLaVA-Mistral, LLaVA-Llama3).
- Visual instruction data pipeline uses GPT-4, cheap to regenerate for new domains.

### LLaVA-1.5 and LLaVA-NeXT

LLaVA-1.5 (October 2023) added:
- Academic task data (VQA, OKVQA, RefCOCO) mixed into instruction tuning.
- Better system prompts.
- 2048 → 32k context.

LLaVA-NeXT (January 2024) added:
- AnyRes: slice high-res images into a 2x2 or 1x3 grid of 336x336 crops plus a global low-res thumbnail. Each crop becomes 576 tokens; ~2880 visual tokens per image total. Major gains on OCR and chart tasks.
- Improved instruction data mix with ShareGPT4V (high-quality GPT-4V captions).
- Stronger base LLMs (Mistral-7B, Yi-34B).

### LLaVA-OneVision

Lesson 12.08 covers OneVision in depth. Short version: same projector, but trained with a curriculum covering single-image, multi-image, and video, sharing a visual token budget across a single model.

### Comparison to Q-Former

| | Q-Former (BLIP-2) | MLP (LLaVA) |
|---|---|---|
| Visual tokens per image | 32 | 576 (base) or 2880 (AnyRes) |
| Trainable parameters | 188M + LM | 40M + LM |
| Stage 1 loss | ITC+ITM+ITG | LM only |
| LLM swappability | Requires retraining | Minimal retraining to swap |
| Multi-image | Awkward | Natural (concatenate) |
| Video | Awkward | Natural (per-frame concatenation) |
| Token budget | Small | Large |

MLP wins on simplicity and token flexibility. Q-Former wins on token budget. By late 2023, token budget stopped being the binding constraint (LLM contexts grew to 32k-128k+), so simplicity dominated.

### Prompt Format

```
A chat between a curious human and an artificial intelligence assistant. The assistant gives helpful, detailed, and polite answers to the human's questions. USER: <image> Describe this image in detail. ASSISTANT: The image shows ...
```

`<image>` is a placeholder token. Before tokenization, it's replaced with 576 visual tokens (2880 under AnyRes). The tokenizer sees a sequence slightly longer than what it trained on, but the LLM handles this novel input because Stage 1 taught it.

### Parameter Economics

LLaVA-1.5-7B breakdown:
- CLIP ViT-L/14 @ 336: 303M (frozen Stage 1, often unfrozen Stage 2).
- Projector (2 linear layers): ~22M trainable.
- Llama-7B: 7B.
- Total: 7.3B parameters. Stage 2 trainable: full 7B + 22M projector.

Stage 2 training cost: ~20 hours on 8xA100. This is the key number — one day, one node, reproducible. This is why LLaVA spread.

## Key Terms

| Term | Common phrasing | What it actually means |
|------|----------------|------------------------|
| Projector | "MLP bridge" | 2-layer MLP with GELU mapping ViT dim to LLM dim |
| Image token | "<image> placeholder" | A prompt marker replaced with N projected visual tokens before inference |
| Visual instruction tuning | "LLaVA Stage 2" | Training on GPT-4-generated (image, instruction, response) triples |
| Stage 1 alignment | "Projector pretraining" | Freezing ViT and LLM, training the projector with LM loss on captions |
| AnyRes | "Multi-crop tiling" | Slicing high-res images into a crop grid and concatenating visual tokens from each crop |
| LLaVA-Instruct | "GPT-4 generated" | 158k instruction-response pairs synthesized from COCO captions + GPT-4 |
| Vision encoder freeze | "Backbone lock" | CLIP weights don't update in Stage 1, sometimes not in Stage 2 either |
| ShareGPT4V | "Better captions" | 1M dense captions generated by GPT-4V for higher-quality alignment |
| VQA | "Visual question answering" | The task of answering free-form questions about an image |
| Prismatic VLMs | "Design space paper" | Karamcheti 2024 ablation that systematically tests projectors and data choices |
